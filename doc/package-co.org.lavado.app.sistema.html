Package co\org\lavado\app\sistema

Classes summary
AccionAuditable Clase que representa aun accion auditable en el sistema. Para ver cuales son las acciones auditables consulte las porpiedades y constantes de esta clase
Auditoria Clase que representa una registro de auditoria en el sistema. El subsistema de auditoria integrado al sistema es bastante exaustivo y robusto. Practicamente se tiene registro de todo cambios hecho en la base de datos conservando rastreo antes y despues de la modificacion
Cargo Clase que representa el cargo de un funcionario en la empresa
CategoriaVariable Clase que define la categoria de una variable de entorno
Clase Clase que define una clase de php usada en el sistema, se tiene por razones de administracion y mantenimiento pues poseemos una tabla en la base de datos que relaciona todas las clases usadas en el sistema y la tabla en la db que mapea
ClaseBase Clase base para todas las clases de instancia de la aplicacion
Combo Clase que define un combo de servicios ofrecido por la empresa. Basicamente se usa para reunir un conjunto de servicios individuales.
Departamento Clase que representa un departamento
Empleado Clase que representa un soldado razo en la empresa. Un empleado quien hace el lavado de carros y presta los servicios en general.
Empresa Clase que hace abtraccion de los datos generales de la empresa.
EstadoCivil Clase que representa el estado civil de una personas natural
EstadoRadicado Clase que representa el estado de un servicios prestado por la empresa
FormaPago Clase que representa una forma de pago, como efectivo, cheque, credito
MarcaAutomotor Clase que representa una marca de automotor, como mazda, akt, honda, etc
Modificacion Clase que representa una modificacion en el sistema. Esta es una clase de apoyo al subsistema de auditoria.
Modulo Clase que representa un modulo en el sistema. UN modulo es una interfaz de usuario independiente, generalmente accedida desde una opcion en el menu del sistema.
ModuloPerfil Clase que representa un modulo incluido en un perfil de usuario.
MovimientoCredito Clase que representa un movimiento credito en el sistema. Los movimientos creditos se manejan para los empleados del sistema y permiten administrar datos como prestamos, ahorros y retiros de dinero hechos por los empleados.
Municipio Clase que representa un municipio.
Pais Clase que representa un pais.
PasoFlujoRadicado Clase que representa un paso en el historial de cambios en el estado de un servicio. Esto permite llevar un control historico de lo que se hace en cada servicio
Perfil Clase que representa un perfil de usuario. Un perfil de usuario es un conjunto logico de modulos que puede ser asignado a un usuario y es la base del sub-sistema de permisos y autorizacion del sistema.
PerfilUsuario Clase que representa un perfil asignado a un usuario.
Persona Clase que representa una persona en el sistema, ya sea natural o juridica. Esta clase es la base para la definicion de usuarios, empleados, clientes, etc
Personal Clase que representa una persona con un cargo especifico dentro de la empresa
Prioridad Clase que representa un priodidad de atencion en el sistema. A los servicios se les puede asignar una prioridad para saber con que urgencia debe ser atendido.
ResolucionTarifa Clase que representa una resolucion de tarifas. Las resoluciones de tarifas son agrupaciones logicas de precios atravez del tiempo, se tiene con el fin de conservar el historico de precios a travez del tiempo.
Rubro Clase que representa un servicios o producto que puede ser cobrado por la empresa.
RubroCombo Clase que representa un rubro agrupado en un combo de servicios
TipoAutomotor Clase que representa un tipo de automotor, como motocicleta, camioneta, etc
TipoIdentificacion Clase que representa un tipo de identificacion, como C.C, N.I.T o T.I entre otros
TipoMovimientoCredito Clase que una causal de movimiento credito, como por ejemplo entradas por ahorros o salidas por prestamos.
TipoRubro Clase que representa un tipo de rubro, permitiendo clasificar lo servicios que presta la empresa en tantas categorias como se deseen
Usuario Clase que representa un usuario del sistema. Los usuarios son los unicos que pueden entrar al sistema y autenticarse emdiante login y password otros
Variable Clase que representa una variable del sistema. Las variables del sistema son fragmentos de informacion canstante que sirven para parametrizar el comportamiento del sistema y en general deben ser entendidas como unidades auxiliares de datos. UNa variable del sistema por ejemplo, puede definir la ubicacion del certificado digital con que se garantiza la integridad de los datos de la db otros
VariableUsuario Clase que representa una variable del sistema especificamente asignada a un usuario del sistema. Esto es util cuando cada usuario del sistema necesita definir para si mismo un comportamiento particular del sistema
